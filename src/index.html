User Management & Permissions

Usage Guide

1. Add User (Single)

Click + Add User, enter a name (e.g. yamada), and click Create. A Linux user user_yamada is created with SSH key pair auto-generated. Download or copy the private key immediately.

2. Add Users (Bulk)

Click Bulk Add, enter multiple names (one per line), and click Create All. SSH keys are generated for each user. Download all keys as a ZIP or individually.

3. Share SSH Key

Send the downloaded private key file to the team member. They save it as ~/.ssh/id_ed25519 and connect: ssh user_yamada@160.16.207.196

4. Permissions

sandbox = All members can read/write. app1app2 = Only the assigned owner can write. Others are read-only.

User Connection Guide (for team members)

  1. Receive private key file from admin (e.g. user_yamada_id_ed25519)
  2. Save to ~/.ssh/ and set permissions:
    chmod 600 ~/.ssh/user_yamada_id_ed25519
  3. Connect to server:
    ssh -i ~/.ssh/user_yamada_id_ed25519 user_yamada@160.16.207.196
  4. Or add to ~/.ssh/config for convenience:
    Host haregake
      HostName 160.16.207.196
      User user_yamada
      IdentityFile ~/.ssh/user_yamada_id_ed25519
    Then just run: ssh haregake

Directory Structure

/var/www/app/

  • sandbox/ - Shared experiment area (all sandbox_users group members can write)
  • app1/ - Production app 1 (owner: user_tanaka)
  • app2/ - Production app 2 (owner: user_sato)

Deploy an App

Admin runs: sudo deploy-app sandbox (or app1, app2)

This rebuilds and restarts the Docker container for that app.
