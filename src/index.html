ユーザー管理・権限設定

使い方ガイド

1. ユーザーを追加する（1人ずつ）

「+ ユーザー追加」ボタンをクリックし、名前（例: yamada）を入力して「作成」を押します。Linuxユーザー user_yamada が作成され、SSH鍵が自動生成されます。秘密鍵をすぐにダウンロードしてください。

2. ユーザーを追加する（一括）

「一括追加」ボタンをクリックし、名前を1行ずつ入力して「一括作成」を押します。全員分のSSH鍵がまとめて生成されます。個別ダウンロードまたは全件ダウンロードが可能です。

3. SSH鍵をメンバーに渡す

ダウンロードした秘密鍵ファイルをメンバーに送ります。メンバーは ~/.ssh/ に保存し、ssh user_yamada@160.16.207.196 で接続できます。

4. 権限の仕組み

sandbox = 全メンバーが読み書き可能な共有実験エリア。app1/app2 = 担当者のみ書き込み可能。sudo/docker = 一般ユーザーには付与されません。

メンバー向け：SSH接続手順

  1. 管理者から秘密鍵ファイル（例: user_yamada_id_ed25519）を受け取る
  2. ~/.ssh/ に保存してパーミッションを設定：
    chmod 600 ~/.ssh/user_yamada_id_ed25519
  3. サーバーに接続：
    ssh -i ~/.ssh/user_yamada_id_ed25519 user_yamada@160.16.207.196
  4. 毎回の入力を省略するには ~/.ssh/config に以下を追加：
    Host haregake
      HostName 160.16.207.196
      User user_yamada
      IdentityFile ~/.ssh/user_yamada_id_ed25519
    以降は ssh haregake だけで接続可能

ディレクトリ構成

/var/www/app/

  • sandbox/ - 共有実験エリア（sandbox権限を持つ全員が読み書き可能）
  • app1/ - 本番用アプリ1（担当者のみ書き込み可）
  • app2/ - 本番用アプリ2（担当者のみ書き込み可）

アプリのデプロイ方法

管理者が以下を実行：sudo deploy-app sandbox（または app1, app2）

Dockerコンテナが再ビルド・再起動されます。
